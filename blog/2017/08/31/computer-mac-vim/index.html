VimでNormalモード切り替え時にIMEをOFFにする、をMacでKarabiner-Elementsを使って実現する

  • B! 11

  • 6

  • 0

  • 0

  • 0

Karabinerが使えなくなってからVimを使っている時に IMEをOFFにしつつノーマルモードに戻る方法が無くて困ってましたが Karabiner-Elementsを使い始めたのをきっかけとして もう一度挑戦してなんとなく上手く行く方法を見つけました。

  • El Capitan以前の方法
  • macOS Sierraになってからの苦闘
  • Karabiner-Elementsを使って実現する
  • まとめ

El Capitan以前の方法

  • Vimを使う上でのIME(日本語入力)の取り扱い

El Capitan以前はKarabinerを使ってIMEをチェックし、IMEの変更まで全て管理していました。

  • Vimを使う上でのIME(日本語入力)の取り扱い with AutoHotKey

ちなみにWindowsではAutoHotkeyを使って同じ様なことをしています。

macOS Sierraになってからの苦闘

で、macOS SierraになってKarabinerが使えなくなって色々となんとか出来ないかやってきました。

  • VimでNormalモード切り替え時にIMEをOFFにする、をMacでKarabiner無しで実現する

まずはAppleScriptでIMEの状態をチェックしてIMEの変更を行えるものを用意し、 そのスクリプトをBetterTouchToolでESCとかに割り当てるというもの。

ショートカットキーの割り当てのアプリはなんでもいいんですがトラックパッドの設定で必ず使うのでついでにBetterTouchToolの中で設定しました。

出てきた問題はAppleScriptを使う方法だとちょっと時間がかかってしまって 勢い良くキーを打ち込むとIMEの変更前に打ち込まれてしまうことがあるということ。

他にも色々細かい問題が出てきて結局使わない事にしました。

  • VimでNormalモード切り替え時にIMEをOFFにする、をMacでKarabiner無しで実現する 2

次が同様にAppleScriptを使いますが今度はAppleScriptでは単にEISUを送るだけのものにします。 (USキーボードのみ使っていてEISUがどこにも無い場合でも送る事は出来ます。)

これを使えばコマンドラインからEISUを送ることができIMEをオフにできます。

これを使って今度は.vimrcの中でInsertLeave時に このコマンドを送る設定をします。

最初の方法ではGNU screenのコピーモード時の問題があるので常にEISUやESCを余計に送る様な設定にしてしまうことができませんでした。 (GNU screenを使ってなかったらそられのキーを連続して常に送れば良い。)

この場合にはVimの中での設定なのでインサートモードからノーマルに戻る時に 常にEISUを送ることは特に問題ありません。 (既に英数状態でも何もしないだけなので。)

この方法は結構上手く行ったのですが、 カーソルキーが使えなくなる問題があったり、 色々工夫してもそれでもまだ遅延があったり して完璧とは言えないものでした。

  • VimでNormalモード切り替え時にIMEをOFFにする、をMacでKarabiner無しで実現する 3

その次に見つけたのがGoogle日本語入力を使ってその中で キー設定を行うというもの。

ESCにキャンセル後IMEを無効化を割り当てると IMEを無効化した上でさらにESCも送れる、というもの。

この方法見つけた時は非常に上手くいって完全に思い通りのことが出来ました。

が、しばらく経った後に違和感を感じて調べてみたらこの方法が効かなくなってました。 (IMEがオンの状態だとESCを押してもIMEがオフになるだけで追加でESCが送られない。)

色々設定を見直しましたが以前と変更したところは無いと思うのでGoogle日本語入力か macOSのアップデートによってだめになってしまったようです。

恐らくGoogle日本語入力の所でキャンセル後IMEを無効化 をしながら更にESCが送られる、という動作が元々予期されないもの(バグ)で、 それが正しく指定の動作だけを行う様になったのかな、と。

ということで残念ながらこの方法は今は使えません。

Karabiner-Elementsを使って実現する

  • Karabiner-Elementsが大分進化してVi Modeとかまで出来る様になっていた
  • KE-complex_modificationsを使ってKarabiner-Elementsの定義を色々作って公開する

Karabiner-Elementsが進化して大分色々出来る様になったので これまで予備的にBetterTouchToolでしのいで居たキーバインド設定をKarabiner-Elementsに全面的に移す事にしました。

やってみた結果、BetterTouchToolだとどうしても出来なかったいくつかの事もまた出来る様になってて素晴らしい限りです。

ただ、まだKarabinerの時にやっていたことで出来ない事がいくつかあって、 そのうちの1つがこのIME問題です。

KarabinerではIMEの情報やIMEを操作する方法がありましたが Karabiner-Elementsにはありません。

追記: 2017/10/30

Karabiner-ElementsでもIMEの情報を得たり直接操作することが出来るようになりました。

  • Karabiner-Elementsで日本語入力状態を把握してキー設定出来る様になった

上の記事では新しいinput_source_ifを使った以前と同じ様な VimでNormalモード切り替え時にIMEをOFFにする を作っています。

追記ここまで

ただ、IMEをオフにするだけであればEISUを送れば良いだけなのでKarabiner-Elementsでも出来ます。

そこで次の様な設定をしてみます。

ESCを押した時に一回ESCを押した後、EISUを送り 再びEISUを送ります。

最初のESCは変換中などにそれを中断させるため、 次のEISUでIMEをオフにし、 もう一度ESCを押すことでVimでインサートモードだった場合にノーマルモードに戻ります。

変換中でなければ最初のESCでノーマルモードに戻りますが 後のESCは無視されるだけです。

これをしてしまうと、本来の変換中にESCを押した時に書き込みをキャンセルしてまたIMEがオンのまま書き続ける、 という動作が出来なくなりますが、 IMEの状態を取れない環境では中途半端にEISUESCだけすると 変換中からだとIMEをオフにした状態でインサートな状態、になり これも中途半端です。

なのでESCを押す時は全部キャンセルしてノーマルに必ず戻る様に。 変換中のキャンセルとかはBackspaceなり⌃-H で一文字づつでも消していけば良いかと。

同様の設定を

にも。

ちょっと副作用が大きいので ターミナルやVim上でのみ有効にしてあります。 (Karabiner-Elementsにあるerb2json.rb中の emacs_key_bindings_exceptionの定義にあたるもの。)

ただ上にも書いたとおりこれをターミナル上で常に有効にしてしまうとGNU screenのコピーモードに入れなくなります。

そこで、~/.screenrcに次の一行を加えます。

bindkey -m ^[ eval ''

bindkey -mはコピーモード時でのキーバインド。

^[は設定するキー(ctrl-[)ですがESCもこの設定で同時に行われます。(GNU screenではこれらは区別されない。)

eval ''で実際に行う設定を入れていますが、通常はeval 'writebuf'とか コマンドを入れますがここではコマンドを空にして何もしないようにしています。 ('<command>'は複数書く事が出来ますが、省略してevalだけにしたりevalも書かなかったりするとエラーが出ました。 何もしたくない場合は空の''をevalに指定すれば良いようです。)

これによって ESCを使ってGNU screenでコピーモードに入っても その後ESCが送られてもコピーモードを保つことが出来ます。

コピーモードを中止したい時は適当に割り当ての無いキーを押せば良いわけですが(ESCも単に割り当ての無いキーだった)、 入る時に使うエスケープキー(自分設定ではQ)の近くに指が残るので、 丁度そのキーコピーモードではは使われてないのそれを押せば良いかな、と。 (自然といままでもQでキャンセルしてることもあったので。)

ちょっと癖が残ってる間はESCで終了させようとして しまいますが全てQにするのはそれほど大変じゃなさそうな感覚。

このGNU screenの設定をすればAppleScriptの最初の試みの所でIMEをチェックせずに ESCとIMEをオフにするコマンドを送ることで遅延時間を減らして行う事も出来るかもしれません。 ただそれでもAppleScriptを呼ぶこと自体が結構負荷あるみたいなので Karabiner-Elementsでやったほうが問題なく出来ます。

ただこのGNU screenの設定をしておけば 今後もMacで何らかの理由でKarabiner-Elementsが使えなくなったときや Windowsでも上手く行かないとき、 またLinuxなんかを使うときでも色々応用して設定が出来そうです。

まとめ

とりあえず今回の設定で再びターミナル内でのVimでも 問題なくVimのインサートモードから Normalモード切り替え時にIMEをOFFにする を1つのキーで行える様になりました。

未だに逆にクセでESC(正確には⌃-[を常に使ってるのでそっち) を2回押してしまう事が多いですが 気をつけて1回で使ってると大分楽に感じます。

今回の設定をまとめると以下の様になります。

Karabiner-Elements complex_modifications rules by rcmdnk へ行ってPersonal rules (@rcmdnk)をImportすると ESC (or Control-[) sends ESC EISU ESCが設定できる様になるので それをEnableすることでも使えます。 (Karabiner-Elementsの新しい設定配布方法とても便利。)
