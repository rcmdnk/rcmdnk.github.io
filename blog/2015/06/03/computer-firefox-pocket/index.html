Firefox 38.0.5で追加されたPocketの機能を無効にする方法

  • B! 2

  • 0

  • 0

  • 0

Firefo 38.0.5の安定版がリリースされ、この中で Pocketというあとで読む系のサービスが統合されました。

ただ、元々アドオンがあってそちらのが使い勝手がよかったり 色々邪魔な所に表示されてる様に感じることもあるので Firefoxの機能としては無効にしたいな、と思った時にすること。

  • Firefox 38.0.5安定版でのPocket
  • Firefox 38.0.5安定版でのPocketを無効にする
  • Firefox 38.0.5安定版でのリーダービューを無効化する

Firefox 38.0.5安定版でのPocket

Firefoxに統合されたというPocketはユーザー側にはメリットが無い? にベータ版の時に見てみた事を書きましたが、 大体これと同じです。

  • FirefoxのアカウントでPocketを使える。(はじめて使いたい場合にはPocketのアカウントを作る必要がない。)
  • Pocketのアドオンを入れてない場合、ツールバーにPocketボタンが出る。(ワンクリック保存)
  • 右クリックからのコンテクストメニューにPocketへ保存するためのメニューが出る。
  • ブックマークにPocketのマイリストを表示が追加される。

といった感じ。 ツールバーのボタンとかはPocketのアドオンを入れてる場合には それを削除するか無効にしないと出ません。

Pocket Blog

安定版を入れてみて気づいたこととしては、 リーダービューと言う状態が加えられていて (もしかしたらベータでもあったかもしれませんが気付かなかった。。。？)、

ツールバーにこんな形の(一番左)のマークが表れ、これをクリックすると

こんな感じの表示になります。 Pocketのサイトで見た感じの見た目。

ただし、このマークが現れるのは特定のサイトだけで、 何故かこのサイトでは表れませんでした。。。 (何かしらの登録なりPocket側で処理されたページだけが出来る？ 一度Pocketに送られたものなら出来るかと思いましたがそうでもないみたい。 後、このマークが表示されても表示できません、と出るページも有りました。)

また、左側には3つのマークがあって、

  • 一番上は元のページに戻る。(もう一度アドレスバーのマークを押しても同じ。)
  • 真ん中のAaは背景を白くしたり黒くしたりフォントを変えたりする。
  • 一番下はポケットに保存？

となってますが、一番下のボタンはマウスオーバーしても何も出ないし 押してみても一瞬赤くなったりするんですが、Pocketに保存されるわけでも無く、 バグ？

と言った感じでなんか不安定(の様に見える)な所もあります。

Firefox 38.0.5安定版でのPocketを無効にする

Firefoxに統合されたというPocketはユーザー側にはメリットが無い? にも書いたとおり、はっきり言って何もメリットがない上、 ブックマークとかに余計な表示が出るだけなので無効にしたくなるもの。

そこで、上に書いた話は

about:configで browser.pocket.enabledをfalseにすれば無効に出来るらしい。

と言うこと。 この値は安定版にもあったのですが、これを試してみたところ、 ツールバーのアイコンが消えるだけでブックマークやコンテクストメニューは 残ったままでした。 アドオンを入れてると元々ツールバーのアイコンはアドオンのものに 取って代わられているので browser.pocket.enabledの値を変えても何も変化がありません。

そこで探してみると下のサイトがヒット。

How to disable Pocket in Firefox - gHacks Tech News

やはり上のやり方は前は出来たけど今はできない、と言った感じで、 無効にするには

  • もし、上のbrowser.pocket.enabledをfalseにしている場合、一旦trueにして再起動する。(ツールバーにFirefoxのPocketアイコン(アドオンのではない)を表示させるため。)
  • Pocketアドオンを入れてる場合、これを一旦無効にする。(これもツールバーにFirefoxのPocketアイコンを表示させるため。これも再起動が必要。上のと一緒に。)
  • ツールバーの右側にあるハンバーガーボタン(三本線の`メニューを開きます`のボタン)を開いて`カスタマイズ`へ行く。
  • ツールバー等のカスタマイズページに行ったらツールバーからPocketのアイコンを ドラッグして左下の**追加のツールと機能**の所へ持って行ってツールバーから削除する。
  • カスタマイズを開かなくてもボタンを右クリックするとツールバーから削除のメニューが出るのでコレをクリックすれば削除完了。

この様にツールバーからボタンを削除すると コンテクストメニューもブックマークのPocketへのリンクも消えます。

またリーダービューからも左のPocketボタンが消えます。

browser.pocket.enabledはtrueのままですが、 表面上はPocketの機能がこれで消えます。

falseにするとアドオン版を入れてない場合には 追加のツールと機能のところからもボタンが消えます。 (アドオンを入れてると元々アドオンのものしか出ない。)

一度削除出来たら、アドオンは有効にしても大丈夫です。 ブックマークとかからは余計なPocketは消えて アドオンの機能だけ使える様になります。

もう一度有効化したい場合には、アドオンを無効にして browser.pocket.enabledをtrueにして Firefox版のボタンをツールバーに追加すればブックマークや コンテクストメニューが表れます。

これで無効に出来てスッキリしますが、上のサイトによると

If you choose to use the Pocket feature, the impact on Firefox’s performance or memory use is minimal, similar to opening Pocket in a new tab in Firefox. For users who choose not to use the feature and remove the button from their toolbar, the impact is reduced even further.

とMozillaは言ってる様です(ただ上の文章が書いてるページが見つけられなかった。。。)。 この文章だけだとPocketのページを開いた時なのか 全てのページについてなのかちょっと良く分かりませんが。 (普通に考えてPocketに関する部分だけだとは思いますが、それも 何がどう良くなるのか良くわからない。。。)

また、Pocket側から今後アドオンのメンテはアクティブにはしないよ、 との発表も。。。

Pocket I already use Pocket for Firefox. What’s…: http://help.getpocket.com/customer/portal/articles/1999057?src=blog

なので今後ずっと使っていこうと思うと いづれアドオンからFirefoxの機能を使わないといけないようになるのかもしれません。

FirefoxもPocket側もユーザーの声に反応して元に戻せる 状態を維持してくれるかもしれませんが。。。

このまま行くのであれば、機能として少なくとも アドレスバー内のPocketのマークで保存、削除が出来る今のアドオンの 機能は入れてほしいな、と。

後、ツールバーから外すと全ての機能がなくなる、とかではなくて、 ツールバーのボタンの配置削除は別にして 全体を無効に出来る機能を付けるのと、 ブックマークのメーニューは外せる様に出来るとかはなんとかしてもらいたい。。。

Firefox 38.0.5安定版でのリーダービューを無効化する

Pocketとは直接関係ないですが、アップデートで入ったもう一つのリーダービューも あえて使う時がない、 というかEvernoteの Clearly と同じ用なもので、こちらの方が全てのサイトで効くので 有用です。

ということで、アドレスバーに余計なボタンがあるのも邪魔だな、と思った場合、 リーダービューを無効にするには about:configで

reader.parse-on-load.enabled

という値を探してこれをfalseにしてFirefoxを再起動します。 これでアドレスバーの余計なボタンも消えます。
