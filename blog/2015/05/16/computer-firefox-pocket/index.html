Firefoxに統合されたというPocketはユーザー側にはメリットが無い?

  • B! 9

  • 0

  • 0

  • 0

  • 0

Firefoxのベータ版の38.0.5がリリースされ、 これに「あとで読む」サービスのPocketが統合された、ということで どんな感じなのか見てみました。

  • Firefoxベータ版
  • 現在のFirefoxでのPocket
  • 38.0.5のPocket

Firefoxベータ版

Firefoxのベータ版はこちらからダウンロードすることが出来ます。

Mozilla Firefox Web Browser — Firefox Developer Edition もしくは Beta を利用して、Firefox の新機能をいち早く試そう — Mozilla

同じページで安定版とアルファ版も上のクルクルで選ぶ事が出来ます。

現在のFirefoxでのPocket

Pocket は「あとで読む」ために保存しておけるサービスの代表的な物の1つですが、 これがFirefoxの機能として入り込んだ、とのこと。

現在Firefox用のものとしてPocketからは アドオン と サービス サービスとしてツールバーにボタンを追加するもの、 また ブックマークレットも提供されています。

アドオン版とサービス版は必要であればそれぞれ別途インストールする必要があります。

アドオン ではツールバーから保存した記事一覧を表示できるボタンを加えたり、

アドレスバーの中にPocketに保存されてるかどうかを示すボタン(かつ、それを押すだけで保存/削除ができる)を加えたりすることが出来ます。

それから以前アドオンで凄く便利だったのがGoogle Readerのページに 「Pocketに保存」ボタンを付けられた事だったんですが(今も設定は残ったまま)、 Google Readerは終わってしまったので今は必要ありません。 Feedlyに付けられればいいんですが、 これはfeedly Proで有料で提供されてるものなので 恐らくPocket側で機能を追加することはないと思います 1。

また、保存する用のショートカットも中にあるんですが、1年位前からこれが上手く動かなくて使えてません (少なくとも手元のMacでもWindowsでも動きません。 そもそもショートカットの設定の所でキー入力しようとしても反応しません。。。一応最新のアップデートは今年の1月位にもあるし、他にこの問題を抱えてるというのを余り見ないのですが。。。)

また、アドオンとは別に、 サービスとしてPocketのサイトから直接インストールできる物 があります。

これはツールバーに表示されて押すだけで保存出来るものです。

押すと登録され、もしやっぱり必要なければ削除もその場でできます。 ただ、アドオン版のアドレスバーに追加されるものとは違い、 既に登録されてるページに行ってもボタンは白黒のままで、 押した瞬間だけ上の様に赤くなるだけなので確認には使えません。

こちらの方がアドオンより新しく出来なものなのですが、 ブックマークレットで十分だし必要ないのでこちらは入れてません。

ということで、Pocket自体は結構使っているので、 これらと何が違って何が嬉しいのかと思って試してみました。

38.0.5のPocket

Mozilla、あとで読むサービス“Pocket”を統合した「Firefox」v38.0.5ベータ版を公開 - 窓の杜

これによるとツールバーに保存用のボタンが入ると共に、 いままでアカウントを持ってない場合はFirefox Syncのアカウントで Pocketにログイン出来るようになるとのこと。

速報： MozillaがPocket（旧Read It Later）をFirefox本体に統合 - Mozilla Flux

さらにここを見るとコンテキストメニューに保存するためのメニューが追加されたり ブックマークの所にPocketへのブックマークみたいのが追加されるとのこと。

実際に入れてみると、まず、右クリックからのコンテキストメニューには メニューが追加されていてここから保存することが出来ました。

また、ブックマーク一覧のところにもPocket用のメニューが。

ただ、これはただPocketのマイメニューへのブックマークなってるだけなので 自分でブックマークすれば良いだけで はっきり言って意味が無いです。

しかもすべてのブックマークのところには出てこないので消せませんでした。 (about:configとかいじれば消えるのでしょうか？環境設定には取り敢えずそれっぽいものはありませんでした。)

追記: 2015/05/17

about:configで browser.pocket.enabledをfalseにすれば無効に出来るらしい。

Mozilla Re-Mix: Firefox 38.0.5でPocketを無効にする方法。

追記ここまで

ツールバーのボタンに関しては、上のページには アドオンを入れてるとアドオン側が無効にされる、とありましたが、 現在のアドオンを入れたプロファイルでFirefox 38.0.5を立ち上げた所、 アドオンは有効で、この新しいボタンは表示されませんでした。

アドオンを無効にして立ち上げると、

な感じでボタンが表示されました。

押してみると

こんな感じでサービスとしてインストールしたPocketのボタンと ほぼ同じ様なものが出ます。

機能としても保存するだけで、保存したページを開いても白黒のままなので サービスの物と機能的には全く一緒です。

なんですが、何故かこのサービスと新しいFirefoxのPocket機能は共存出来ます。。。

一方、アドオンは有効にするとこのFirefoxのボタンは消えてしまうので 共存出来ません。 ちょっと何を意図してるのか意味が分からない部分があります。。。

取り敢えず使ってみた感想としては、全く必要の無い統合、と言う感じです。

FirefoxではAndroidにはリーディングリストを使って後で読む機能があったりしましたが、 上の記事によるとこれをデスクトップ版に持ってくる様な開発等をしてたものの、 この機能をPocketで、と言うことになったみたいです。

ただ、Android版もベータ版が出てますが、こちらは Pocketの統合はまだみたいです。

Firefox側は後で読む機能を簡単に取り入れられ、 Pocket側はFirefoxからより多くのユーザーを獲得、 ということみたいですが、この統合は微妙なところ。

Firefoxとしての機能を簡単にオフに出来る様に(というかデフォルトでオフで 使いたい人だけオンに出来るように) してくれないかな、という気がします。 まあ、わざわざオンにするような人はアドオンやサービス使うので 必要ない、と言うことになってしまいますが。。。

唯一、これまでPocketを使ってなかった人が 改めてアカウントを自分で作らなくても Firefox Syncのアカウントで簡単に使えるようになる、ということくらいが ユーザー側のメリットでしょうか？

Redditなんかでもこれはダメだろ、的な雰囲気で結構紛糾してたりして、 やっぱりユーザー側には何もメリットが無いどころか迷惑なだけの統合な感じがしてます。。。

Firefox Beta now integrates Pocket a proprietary, closed source service. : linux

まあ、まだベータなので、実際に安定版になる時にどうなってるかは分かりませんが。
